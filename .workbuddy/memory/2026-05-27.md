# 2026-05-27

## Infinite-Canvas 项目启动脚本修复

### 问题
- 用户系统没有安装 Python，双击 `安装依赖.bat` 报 "Python not found"
- bat 文件中文乱码（UTF-8 without BOM vs cmd 默认 GBK）
- PowerShell 脚本 `setup_python.ps1` 变量名被 bash 吃掉（$ 符号被 shell 替换）

### 修复内容
1. **新增 `setup_python.ps1`**：自动从 python.org 下载 Python 3.13.12 嵌入版，解压到 `python/` 文件夹，启用 site-packages，安装 pip
2. **重写 `安装依赖.bat`**：加入自动检测 → 调用 ps1 下载 Python → 安装依赖的完整流程；保存为 UTF-8 with BOM + `chcp 65001`
3. **重写 `run.bat`**：同样 UTF-8 with BOM + `chcp 65001`
4. **修正 `README.md`**：端口 5000→3000，启动命令 `app.py`→`main.py`

### 测试结果
- 成功启动 FastAPI 服务（端口 3000），后台运行中
- `安装依赖.bat` 能正常调用 PowerShell 下载 Python，无乱码无报错

### 技术要点
- Windows cmd 读取 bat 文件默认用 GBK，必须用 UTF-8 with BOM 或 `chcp 65001`
- bash 执行 Python -c 时 `$var` 会被 shell 变量替换，写文件要用 Write 工具或转义
- Python 嵌入版（embeddable）需要修改 `python313._pth` 取消 `#import site` 注释
